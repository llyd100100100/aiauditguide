Phase 1(프로토타입) 단계에서 제안드린 기술 스택(Python, Streamlit, Pandas, Microsoft Presidio)은 **"개발 속도(Time-to-Market)"**와 **"검증 효율성(Validation Efficiency)"**을 최우선으로 고려한 조합입니다.

이 조합을 선택한 구체적인 기술적, 전략적 이유는 다음과 같습니다.

1. 언어: Python (파이썬)
데이터 및 AI 생태계의 표준: 2025년 기준, 데이터 분석과 AI 개발에서 가장 압도적인 점유율을 가진 언어입니다. Pandas, Scikit-learn 같은 데이터 분석 라이브러리와 OpenAI, LangChain 등 AI 라이브러리가 Python을 1순위로 지원하므로, 문제가 생겼을 때 참고할 자료(Reference)가 가장 많습니다.

보안 감사 자동화 적합성: Python은 네트워크 스캔, 로그 파싱, 취약점 점검 자동화에 강점이 있어 보안 엔지니어링 분야에서도 널리 쓰입니다. 즉, 추후 시스템이 커져도 언어를 바꿀 필요가 없습니다.

2. 프론트엔드: Streamlit (스트림릿)
가장 고민이 많으실 부분일 텐데, React나 Vue.js 같은 전문 웹 프레임워크 대신 Streamlit을 선택한 이유는 명확합니다.

HTML/CSS/JS 없이 개발 가능: Streamlit은 100% Python 코드만으로 웹 화면을 만듭니다. 프론트엔드 개발자가 따로 없어도 데이터 엔지니어 혼자서 1~2일 만에 그럴듯한 웹 대시보드를 만들 수 있습니다.

데이터 대시보드에 특화: 경쟁 도구인 Gradio가 'AI 모델의 입력/출력 시연'에 특화되어 있다면, Streamlit은 차트, 데이터 테이블, 사이드바 메뉴 등 **'데이터 분석 대시보드'**를 구성하는 데 훨씬 강력한 기능을 제공합니다. Audit 분석 결과는 텍스트와 통계가 섞여 있으므로 Streamlit이 훨씬 적합합니다.

3. 데이터 처리: Pandas (판다스)
로그 파일 처리의 사실상 표준: 엑셀, CSV, JSON 등 다양한 형식의 Audit 로그를 단 한 줄의 코드로 불러오고(pd.read_csv), 필터링하거나 형식을 변환하는 데 최적화되어 있습니다.

전처리 효율성: AI에게 데이터를 보내기 전에 불필요한 공백을 제거하거나, 날짜 형식을 통일하는 등의 '전처리 작업'을 가장 빠르고 효율적으로 수행합니다.

4. 보안/전처리: Microsoft Presidio
이 프로젝트의 핵심 리스크인 **'데이터 유출'**을 막기 위해 선택했습니다.

검증된 PII(개인정보) 식별 엔진: 단순히 정규표현식(Regex)으로 이메일을 찾는 것을 넘어, 문맥을 파악하여 이름, 전화번호, IP 주소 등을 식별하는 NLP(자연어 처리) 기반 엔진입니다.

오픈소스 및 커스터마이징: Microsoft가 관리하는 오픈소스로 신뢰할 수 있으며, 한국 사용자의 특성에 맞춰 주민등록번호 패턴 등을 추가하거나 수정하기 용이합니다. 프로토타입 단계에서 '보안성'을 경영진에게 어필할 때 가장 강력한 근거가 됩니다.

5. AI 연동: Direct API (vs LangChain)
초기 단계에서는 LangChain 같은 복잡한 프레임워크 대신 OpenAI/Anthropic의 API를 직접 호출하는 방식을 권장합니다.

디버깅 용이성: LangChain은 기능이 강력하지만 추상화(Abstraction)가 많이 되어 있어, 초기에 에러가 발생하면 원인을 찾기 힘들 수 있습니다. 프로토타입에서는 **'입력 프롬프트 -> API 호출 -> 결과 수신'**의 과정이 투명하게 보이는 것이 검증에 유리합니다.

구조화된 출력(Structured Output): 최신 API들은 JSON 모드를 강력하게 지원하므로, 별도의 라이브러리 없이도 AI의 답변을 프로그램이 이해하기 쉬운 데이터 포맷으로 받을 수 있습니다.

요약: 왜 이 조합인가요?
**"개발자 1명이 1주일 안에, 보안이 적용된 웹 기반 AI 분석 도구를 만들어 시연할 수 있는 유일한 조합"**이기 때문입니다.

Frontend: Streamlit (디자인 시간 절약)

Backend: Python + Pandas (데이터 처리 속도 확보)

Security: Presidio (개인정보 유출 원천 차단)

이 구성으로 빠르게 가능성을 확인(Phase 1)한 후, 예산을 확보하여 하드웨어 도입과 전문화 단계(Phase 2~)로 넘어가는 것이 가장 리스크가 적은 전략입니다.
