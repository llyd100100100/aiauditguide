# AI-Audit-Reviewer 프로젝트 요약

## 1. 프로젝트 개요
**AI-Audit-Reviewer**는 감사 추적(Audit Trail) 로그 분석을 자동화하는 보안 중심의 AI 시스템입니다. 이 시스템은 사람의 눈으로 찾기 힘든 데이터 무결성 위반, 오류, 비정상적인 접근 패턴을 탐지하여 품질 보증(QA) 및 보안 팀을 지원합니다.

이 시스템은 **FDA 21 CFR Part 11** 및 **ALCOA+** 원칙과 같은 엄격한 규제를 준수하도록 설계되어, 제약/바이오 등 규제 산업 분야에 적합합니다.

## 2. 핵심 목표
- **자동 이상 탐지**: 감사 로그에서 오류, 데이터 삭제, 의심스러운 활동을 자동으로 식별합니다.
- **문맥 기반 분석 (Context-Aware)**: 단순 키워드 매칭을 넘어, "업무 시간 외 비정상 접근"과 같은 복합적인 위협 시나리오를 이해합니다.
- **데이터 프라이버시 (Zero Trust)**: 민감한 개인정보(PII)는 클라우드 AI로 전송되기 *전에* 완벽하게 마스킹(가림 처리)됩니다.
- **설명 가능성 (Explainability)**: AI가 탐지한 모든 이상 징후에 대해 구체적인 로그 라인 번호와 이유를 설명합니다.

## 3. 주요 기능 (Phase 1 프로토타입)
- **다중 포맷 지원**: CSV, Excel, TXT, JSON 등 다양한 로그 파일을 업로드할 수 있습니다.
- **PII 방화벽**: Microsoft Presidio를 사용하여 이름, IP, ID 등을 자동으로 감지하고 마스킹합니다.
- **AI 분석**: **Google Gemini (Flash)**를 활용하여 *익명화된* 로그의 보안 위험을 분석합니다.
- **대시보드**: 웹 기반 UI(Streamlit)를 통해 파일을 간편하게 업로드하고 분석 결과를 시각적으로 확인합니다.

## 4. 개발 로드맵
이 프로젝트는 "바이브 코딩(Vibe Coding)" 전략에 따라 5단계로 진화합니다.

1.  **Phase 1: 프로토타입 (현재 단계)**
    *   **기술 스택**: Python, Streamlit, Pandas, Presidio, Gemini API.
    *   **목표**: 개인정보 유출 없이 AI가 감사 로그에서 유의미한 오류를 찾아낼 수 있음을 검증.
2.  **Phase 2: 구조 학습 (Structural Learning)**
    *   Vector DB(RAG)를 도입하여 회사의 SOP(표준운영절차)와 정상 패턴을 학습.
3.  **Phase 3: 휴먼 인 더 루프 (Human-in-the-Loop)**
    *   사용자가 AI의 분석 결과를 승인하거나 거부하는 피드백 기능을 추가하여 정확도 개선.
4.  **Phase 4: 자동 수집 (Automated Ingestion)**
    *   서버 폴더 자동 감시 및 실시간 알림 시스템 구축.
5.  **Phase 5: 전문가 시스템 (On-Premise)**
    *   보안이 완벽한 로컬 하드웨어(NVIDIA RTX 6000 Ada) 및 로컬 LLM(Llama 3/Qwen 2.5)으로 이관하여 폐쇄망 운영.
